# Data

Versioned data snapshots live here.

- `rdap-bootstrap/`: IANA RDAP bootstrap snapshots generated by `scripts/update-data/update-rdap-bootstrap.sh`, plus `extra.json` for reviewed RDAP routes missing from IANA bootstrap data.
- `registrars/`: ICANN accredited registrar CSV generated by `scripts/update-data/update-registrars.sh`.
- `brands/`: registrar and nameserver brand map kept as reviewable JSON and synchronized into the API embedded snapshot by `scripts/update-data/sync-brand-map.sh`.
- `enrichment/`: optional local Pricing/Moz datasets synchronized into the API embedded snapshot by `scripts/update-data/sync-enrichment.sh`.
- `whois-servers/`: WHOIS server snapshots synchronized from the local reviewable data into the API embedded snapshot by `scripts/update-data/sync-whois-servers.sh`.
- `public-suffix/`: full Public Suffix List snapshot generated by `scripts/update-data/update-public-suffix.sh`, plus a small `extra.dat` overlay synchronized by `scripts/update-data/sync-public-suffix.sh`.

Run `pnpm test:data` after changing any file in this directory. The validator checks manifest hashes, parsability, embedded snapshot sync, and critical routing coverage for high-value public suffixes and WHOIS server overrides.

Runtime checks the mounted data directory first, then uses the embedded RDAP bootstrap snapshot, then falls back to live IANA bootstrap with in-memory caching when a query is not covered.
Registrar enrichment checks the mounted data directory first, then uses the embedded registrar snapshot.
Brand enrichment checks the mounted data directory first, then uses the embedded brand map snapshot.
Pricing/Moz enrichment checks mounted `enrichment/*.json` first, then uses embedded optional datasets. These enrichers do not call external APIs during lookup.
WHOIS lookup checks the mounted WHOIS server directory first, then uses embedded server snapshots, then falls back to a small built-in minimum set.
Domain normalization uses a mounted or embedded `public-suffix/public_suffix_list.dat` snapshot when available, then applies `public-suffix/extra.dat`. If no full snapshot is present, it falls back to `golang.org/x/net/publicsuffix` plus the overlay.

For single-file compose deploys, put files beside `docker-compose.yml`:

```text
data/
  rdap-bootstrap/
    dns.json
    ipv4.json
    ipv6.json
    asn.json
  registrars/
    icann-accredited-registrars.csv
  brands/
    brand-map.json
  enrichment/
    pricing.json
    moz.json
  whois-servers/
    iana.json
    extra.json
  public-suffix/
    public_suffix_list.dat
    extra.dat
```

`scripts/update-data/update-rdap-bootstrap.sh` writes reviewable copies here and also refreshes the embedded API snapshots in `services/lookup-api/internal/data/rdapbootstrap/snapshots`.
`scripts/update-data/update-registrars.sh` writes reviewable copies here and also refreshes the embedded API snapshot in `services/lookup-api/internal/data/registrars/snapshots`.
`scripts/update-data/update-public-suffix.sh` writes the official PSL snapshot here and also refreshes the embedded API snapshot in `services/lookup-api/internal/data/publicsuffixes/snapshots`.
`scripts/update-data/sync-brand-map.sh` copies the reviewable brand map into `services/lookup-api/internal/data/brandmap/snapshots`.
`scripts/update-data/sync-enrichment.sh` copies the reviewable optional enrichment datasets into `services/lookup-api/internal/data/enrichment/snapshots`.
`scripts/update-data/sync-whois-servers.sh` copies the reviewable WHOIS server maps into `services/lookup-api/internal/data/whoisservers/snapshots`.
`scripts/update-data/sync-public-suffix.sh` copies the reviewable PSL overlay into `services/lookup-api/internal/data/publicsuffixes/snapshots`.

The scheduled data update workflow runs the same validator before committing snapshot changes.
